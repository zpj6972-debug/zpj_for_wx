# UI 风格规约 — Mini Program Framework

> 本规约适用于基于本框架开发的所有微信小程序页面与组件。

---

## 1. 设计原则

- **简洁清晰**：界面元素精简，信息层级明确
- **一致性**：所有页面遵循统一的色彩、间距、字体和交互规范
- **开发者友好**：组件样式易于覆盖和扩展，使用语义化 token
- **平台原生感**：尊重微信小程序设计惯例，避免与系统控件冲突

---

## 2. 色彩系统

### 2.1 主色调

| Token | 值 | 用途 |
|-------|-----|------|
| `--primary` | `#4670E7` | 主按钮、链接、激活状态、关键图标 |
| `--primary-dark` | `#3A5DD0` | 按钮按下态、主色 hover |
| `--primary-light` | `#E8EEFD` | 主色背景、轻量标签背景 |

### 2.2 中性色

| Token | 值 | 用途 |
|-------|-----|------|
| `--text-primary` | `#1A1A1A` | 标题、正文 |
| `--text-secondary` | `#666666` | 次要文字、说明文字 |
| `--text-tertiary` | `#999999` | 占位符、禁用态文字 |
| `--border` | `#E5E5E5` | 分割线、卡片边框 |
| `--surface` | `#FFFFFF` | 卡片、浮层背景 |
| `--background` | `#F1F6FF` | 页面全局背景 |

### 2.3 功能色

| Token | 值 | 用途 |
|-------|-----|------|
| `--success` | `#34C759` | 成功状态、通过提示 |
| `--warning` | `#FF9500` | 警告状态、待处理 |
| `--error` | `#FF3B30` | 错误状态、删除操作 |
| `--info` | `#5AC8FA` | 信息提示、辅助说明 |

### 2.4 暗色模式（预留）

暗色模式使用降饱和度的变体，而非简单反色：
- 背景：`#0F172A`
- 表面：`#1E293B`
- 主文字：`#F1F5F9`
- 次文字：`#94A3B8`

## 3. 字体排版

### 3.1 字体栈

```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, 'Noto Sans', sans-serif;
```

> 微信小程序使用系统字体，不引入外部字体文件。

### 3.2 字号规范

| 层级 | 大小 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| Display | 48rpx | 700 | 1.2 | 空状态大标题 |
| H1 | 36rpx | 700 | 1.3 | 页面大标题 |
| H2 | 32rpx | 600 | 1.4 | 区块标题 |
| H3 | 28rpx | 600 | 1.4 | 卡片标题、列表标题 |
| Body | 28rpx | 400 | 1.6 | 正文内容 |
| Caption | 24rpx | 400 | 1.5 | 辅助说明、时间戳 |
| Small | 20rpx | 400 | 1.4 | 标签、徽章文字 |

### 3.3 字体规则

- 最小正文字号不低于 24rpx（约 12px），避免 iOS 自动缩放
- 标题使用 semibold(600) 或 bold(700)，正文使用 regular(400)
- 禁止使用小于 20rpx 的字体
- 文字颜色不使用纯黑 `#000000`，使用 `--text-primary` `#1A1A1A`

## 4. 间距系统

基于 **8rpx 基数** 的间距体系：

| Token | 值 | 用途 |
|-------|-----|------|
| `space-1` | 8rpx | 图标与文字间距、紧凑内边距 |
| `space-2` | 16rpx | 小卡片内边距、按钮内边距 |
| `space-3` | 24rpx | 标准卡片内边距、表单字段间距 |
| `space-4` | 32rpx | 区块间距、页面水平边距 |
| `space-5` | 48rpx | 大区块间距、分隔间距 |
| `space-6` | 64rpx | 页面顶部/底部安全间距 |

### 4.1 页面边距

- 移动端页面水平边距：**40rpx**（左右各 20rpx）
- 卡片内边距：**24rpx ~ 32rpx**
- 元素之间最小间距：**16rpx**

---

## 5. 圆角规范

| Token | 值 | 用途 |
|-------|-----|------|
| `radius-sm` | 8rpx | 小按钮、标签、输入框 |
| `radius-md` | 12rpx | 标准按钮、卡片 |
| `radius-lg` | 16rpx | 大卡片、弹窗 |
| `radius-xl` | 24rpx | 底部弹窗、大容器 |
| `radius-full` | 9999rpx | 胶囊按钮、头像 |

---

## 6. 阴影与 elevation

微信小程序中阴影使用谨慎，避免性能问题：

| 层级 | 阴影 | 用途 |
|------|------|------|
| Elevation-1 | `0 2rpx 8rpx rgba(0,0,0,0.06)` | 轻微浮起（标签、小按钮） |
| Elevation-2 | `0 4rpx 16rpx rgba(0,0,0,0.08)` | 卡片、列表项 |
| Elevation-3 | `0 8rpx 24rpx rgba(0,0,0,0.10)` | 弹窗、浮层 |

> 优先使用边框和背景色区分层级，阴影仅用于需要强调浮起感的元素。

## 7. 组件规范

### 7.1 按钮

**主按钮**
- 背景：`--primary` `#4670E7`
- 文字：白色，28rpx，字重 500
- 高度：88rpx
- 圆角：`radius-md` 12rpx
- 按下态：背景变为 `--primary-dark` `#3A5DD0`
- 禁用态：背景 `#B8C8F5`，文字白色

**次按钮**
- 背景：透明
- 边框：1rpx solid `--primary`
- 文字：`--primary`
- 按下态：背景 `--primary-light`

**文字按钮**
- 背景：透明
- 文字：`--primary`
- 按下态：文字透明度 0.7

### 7.2 卡片

- 背景：`--surface` `#FFFFFF`
- 圆角：`radius-lg` 16rpx
- 内边距：24rpx ~ 32rpx
- 阴影：`Elevation-2`
- 统一使用 8rpx 为单位的 margin/padding

### 7.3 输入框

- 高度：88rpx
- 背景：`#F5F7FA`
- 圆角：`radius-md` 12rpx
- 内边距：24rpx
- 边框：默认无，聚焦时 2rpx solid `--primary`
- 占位符颜色：`--text-tertiary`

### 7.4 导航栏

- 背景：`--primary` `#4670E7`
- 高度：动态计算（适配不同机型胶囊按钮位置）
- 标题：白色，28rpx，居中
- 返回按钮：白色箭头图标
- 状态栏由系统处理，导航栏内容区域从胶囊按钮下方开始

---

## 8. 图标规范

- **禁止使用 emoji 作为图标**，统一使用 iconfont 或 SVG
- 图标尺寸规范：24rpx / 32rpx / 48rpx
- 图标颜色跟随上下文：主按钮内用白色，正文旁用 `--text-secondary`
- 描边风格统一：线框图标使用 1.5rpx 描边，实心图标保持填充
- 触摸目标：图标点击区域不小于 44rpx × 44rpx

---

## 9. 布局规范

### 9.1 页面结构

```
Page
├── Custom NavBar（固定顶部）
├── ScrollView / View（内容区）
│   ├── 区块 1
│   ├── 区块 2
│   └── ...
└──（可选）Bottom Bar / TabBar
```

### 9.2 安全区域

- 顶部：预留导航栏高度（通过 `app.globalData.navHeight` 动态获取）
- 底部：iPhone X 及以上机型底部安全区域 34rpx（通过 `env(safe-area-inset-bottom)` 或 padding 处理）
- 内容不贴边：最小水平边距 20rpx

### 9.3 响应式断点

微信小程序以 750rpx 为设计基准：

| 设备类型 | 宽度范围 | 布局策略 |
|----------|----------|----------|
| 小屏手机 | ≤375px | 单列，标准间距 |
| 大屏手机 | 376~430px | 单列，适度放宽间距 |
| 平板 | ≥431px | 可双列，最大内容宽度 690rpx 居中 |

## 10. 交互规范

### 10.1 触摸反馈

- 按钮按下：背景色变深或透明度降至 0.85，过渡时间 150ms
- 列表项按下：背景变为 `--primary-light`，过渡时间 100ms
- 卡片按下：轻微缩放至 0.98，过渡时间 200ms

### 10.2 动画时长

| 动画类型 | 时长 | 缓动函数 |
|----------|------|----------|
| 微交互（按钮、图标） | 150ms | ease-out |
| 页面过渡 | 250ms | ease-in-out |
| 弹窗出现 | 200ms | ease-out |
| 弹窗消失 | 150ms | ease-in |
| 骨架屏 shimmer | 1200ms | linear infinite |

### 10.3 加载状态

- 小于 300ms：无加载提示
- 300ms~1s：按钮内显示 loading 图标
- 大于 1s：显示骨架屏或全屏 loading

### 10.4 手势

- 支持系统返回手势（iOS 侧滑返回）
- 列表项支持左滑操作（如删除）时需提供明确的视觉提示
- 禁止阻断系统手势（控制中心、底部上滑等）

---

## 11. 可访问性

- 所有图片必须设置 `alt` 或 `aria-label`
- 纯图标按钮必须设置 `aria-label` 说明功能
- 正文与背景对比度不低于 4.5:1
- 支持系统字体大小调整（避免固定尺寸导致截断）
- 减少动态效果偏好（`prefers-reduced-motion`）下禁用非必要动画

---

## 12. 文件组织

样式按职责分离：

```
styles/
├── variables.wxss    # 颜色、间距、圆角 token
├── mixins.wxss       # 通用样式混合（可选）
├── utilities.wxss    # 工具类补充（框架级）
└── tailwind.wxss     # TailwindCSS 入口
```

页面级样式：
- 每个页面独立 `.wxss` 文件
- 优先使用 TailwindCSS 工具类，复杂样式写入页面 wxss
- 禁止在页面中硬编码色值，统一引用 token

---

## 13. Anti-Patterns（禁止项）

- [ ] 禁止混合使用 px 和 rpx，统一使用 rpx
- [ ] 禁止在代码中写死色值（如 `#4670E7`），必须使用 token
- [ ] 禁止使用 emoji 替代图标
- [ ] 禁止正文小于 24rpx
- [ ] 禁止按钮高度低于 72rpx
- [ ] 禁止触摸目标小于 44rpx × 44rpx
- [ ] 禁止水平滚动（表格等特殊情况除外）
- [ ] 禁止动画时长超过 500ms
- [ ] 禁止页面无 loading/空状态处理
- [ ] 禁止阻断系统返回手势
